# 多学科交互学习网站结构设计

## 目标

网站需要从单个“浮力页面”升级为可扩展的学习平台。核心路径是：

```text
学科 -> 知识点 -> 交互讲解 / 实验 / 例题 / 练习 / 复盘
```

首期仍以初中物理“浮力”为完整功能样板，同时提供物理“速度”和数学知识点入口，验证平台结构能够容纳多学科扩展。

## 信息架构

### 顶层：学科

- 数学
- 物理

后续可扩展：化学、生物、地理等。

### 第二层：知识点

物理首批：

- 浮力：完整实现
- 速度：提供知识点入口和交互预览
- 欧姆定律：规划入口
- 凸透镜成像：规划入口

数学首批：

- 一次函数
- 相似三角形
- 方程与不等式

### 第三层：知识点学习页

每个知识点统一包含：

- 学习目标
- 核心概念
- 交互实验或动态模型
- 典型例题
- 做题思路
- 分层练习
- 结果复盘

## 当前实现策略

为了保持静态站点简单可靠，本阶段使用单页应用式布局：

- 左侧或顶部显示学科切换。
- 学科下方显示知识点列表。
- 主区域展示当前知识点内容。
- 浮力为完整内容。
- 速度为交互预览。
- 数学知识点先作为可扩展入口，避免用户误以为已经完整开放。

## 交互设计

### 平台导航

- 学科使用 segmented control。
- 知识点使用 topic rail / topic cards。
- 当前知识点有状态标识：完整、预览、规划中。

### 浮力学习

保留现有完整功能：

- 浮力实验台
- 实时计算
- 当前趋势与最终状态
- 单题练习流
- 总复盘

### 速度预览

速度知识点用于验证多知识点结构，提供轻量交互：

- 调整路程和时间。
- 实时计算速度。
- 显示公式 v = s / t。

## 验收标准

- 页面第一屏能看出这是多学科学习平台，而不是单个浮力页面。
- 可以在数学/物理之间切换。
- 可以在物理下选择浮力和速度。
- 浮力原有实验台和练习功能可继续使用。
- 页面结构、样式和脚本有测试覆盖。
